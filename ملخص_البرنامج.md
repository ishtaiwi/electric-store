# Electrical Store — ملخص البرنامج الكامل

> نظام إدارة متجر أدوات ومستلزمات كهربائية  
> **اسم الحزمة:** `electrical_store`  
> **الإصدار:** `1.0.0+1`  
> **الناشر:** Osama  
> **المنصة الأساسية:** Windows Desktop (x64)

---

## 1. نظرة عامة

**Electrical Store** تطبيق سطح مكتب مبني بـ **Flutter** لإدارة محل أدوات كهربائية بشكل متكامل. يجمع بين نقطة البيع (POS)، المخزون، الفواتير، العملاء والموردين، المصروفات، التقارير، والنسخ الاحتياطي في واجهة واحدة تدعم العربية والإنجليزية مع اتجاه RTL والعملة **₪ (شيكل)**.

### الهدف من البرنامج

تسهيل تشغيل المحل يومياً من شاشة واحدة: بيع سريع، تتبع مخزون، متابعة ديون العملاء، إدارة الموردين، طباعة الفواتير والكشوف، ونسخ احتياطي للبيانات بدون الحاجة لسيرفر سحابي.

## 3. الشاشات والوحدات الرئيسية

التنقل عبر شريط جانبي (**NavigationRail**) مع **11 تبويباً**:

1. **لوحة التحكم (Dashboard)**
2. **المبيعات / نقطة البيع (Sales)**
3. **كل المبيعات (All Sales)**
4. **المنتجات (Products)**
5. **العملاء (Customers)**
6. **الفواتير (Invoices)**
7. **المصروفات (Expenses)**
8. **الموردين (Suppliers)**
9. **قوائم الأسعار (Price Lists)**
10. **النسخ الاحتياطي (Backup)**
11. **الإعدادات (Settings)**

### شاشات فرعية إضافية

- تسجيل الدخول (Login)
- كشف حساب العميل (Customer Account Statement / Ledger)
- تقرير ربح كشف الحساب (Account Ledger Profit)

## 4. الميزات بالتفصيل

### 4.1 تسجيل الدخول والصلاحيات

- دخول باسم مستخدم وكلمة مرور
- كلمات المرور محمية بـ **SHA-256 + Salt**
- ترحيل تلقائي لكلمات المرور القديمة (نص صريح → تجزئة)
- ثلاثة أدوار: **Admin** / **Manager** / **Cashier**
- تغيير كلمة المرور مع التحقق من القديمة
- تسجيل أحداث الدخول والخروج والفشل في سجل التدقيق (Audit)

### 4.2 لوحة التحكم

- بطاقات وصول سريع لكل الأقسام
- تنبيهات: مخزون منخفض، نفاد مخزون، إجمالي ديون العملاء
- ملخص اليوم: مبيعات، ربح، عدد الفواتير
- اختصارات: بيع جديد، إضافة منتج، إضافة عميل، عرض فواتير
- زر عائم للمساعد الذكي (AI Chatbot)

### 4.3 إدارة المنتجات والمخزون

- إضافة / تعديل / حذف منتجات
- بحث بالاسم أو الباركود أو الملاحظات
- فلتر للمخزون المنخفض
- تعديل مخزون: إدخال / إخراج / إرجاع مع سبب وسجل
- ترقيم صفحات (50 منتج لكل صفحة)
- حساب الربح وهامش الربح لكل منتج
- ربط اختياري بمورد (`supplier_id`)

**حقول المنتج:** الاسم، الباركود، الكمية، سعر البيع، سعر التكلفة، ملاحظات، المورد، الحد الأدنى للمخزون، آخر تحديث.

### 4.4 نقطة البيع (POS)

- بحث باركود مع تركيز تلقائي
- بحث ذكي (Fuzzy / N-gram / صوتي عربي)
- سلة مشتريات: كمية، سعر مخصص، حذف بند
- منتجات غير مسجلة (بدون خصم من المخزون)
- خصم إجمالي على الفاتورة
- اختيار عميل أو بيع عابر (Walk-in)
- طرق الدفع: نقد / بطاقة / جزئي / آجل (الآجل يتطلب عميل)
- إتمام البيع بمعاملة قاعدة بيانات واحدة (فاتورة + بنود + تحديث مخزون)
- رقم فاتورة بـ UUID
- إلغاء بيع يعيد المخزون تلقائياً

### 4.5 سجل كل المبيعات

- جدول تاريخ المبيعات مع بحث وترقيم
- انتقال لتقرير ربح كشف الحساب

### 4.6 العملاء وكشف الحساب

- إدارة كاملة للعملاء
- فلتر حسب الديون
- كشف حساب محاسبي (مدين / دائن)
- أنواع القيود: فاتورة، دفعة، خصم حساب، تسوية، رصيد افتتاحي
- دفعات: نقد / شيك / خصم + رقم شيك
- تسجيل بيع مباشرة من كشف الحساب
- طباعة / حفظ PDF للكشف مع فلاتر
- إنشاء فاتورة من سياق العميل

### 4.7 الفواتير

- فلترة بالتاريخ وطريقة الدفع والبحث
- حالات: مدفوعة / جزئية / غير مدفوعة / زائدة الدفع
- تحديث المدفوع والملاحظات
- حذف، طباعة، وحفظ PDF

### 4.8 المصروفات

- إضافة وتعديل وحذف مصروفات
- فئات: مرافق، إيجار، رواتب، مستلزمات، صيانة، شحن، أخرى
- فلتر تاريخ / فئة مع إجمالي بالشيكل

### 4.9 الموردين

- إدارة الموردين مع مرفقات (PDF / صور)
- فواتير موردين ودفعات (نقد / شيك)
- متابعة الأرصدة والمستحقات
- حذف متسلسل للفواتير والدفعات والملفات المرتبطة

### 4.10 قوائم الأسعار

- إنشاء قوائم أسعار وبنودها
- ربط اختياري بعميل
- طباعة وحفظ PDF

### 4.11 النسخ الاحتياطي والاستعادة

- نسخ احتياطي بصيغة ZIP
- استعادة من ملف قاعدة بيانات `.db`
- عرض تاريخ النسخ وحذفها
- تحذير قبل الاستعادة

### 4.12 الإعدادات

- بيانات المحل (اسم، ضريبة، عملة…)
- تبديل اللغة فورياً (عربي / إنجليزي)
- إدارة المستخدمين (إضافة / حذف؛ منع حذف آخر Admin)
- معلومات النظام (إصدار، قاعدة البيانات، المنصة)

### 4.13 المساعد الذكي (Chatbot)

- يعمل محلياً بدون إنترنت
- فهم أوامر عربية وإنجليزية (Regex NLU)
- حوالي 14 نية استعلام تستخرج بيانات مباشرة من SQLite
- مناسب لأسئلة سريعة عن المبيعات والمخزون والديون

### 4.14 البحث الذكي

- مطابقة دقيقة + تقريبية (Fuzzy)
- N-gram وتطبيع حروف عربية
- مرادفات خاصة بمجال الكهرباء

### 4.15 التقارير والتحليلات

- إحصائيات لوحة التحكم (اليوم / الشهر) مع كاش ~30 ثانية
- تقرير المبيعات اليومية
- تقرير الأرباح (إيراد، إجمالي ربح، مصروفات، صافي، هامش %)
- تقرير المخزون (متوفر / منخفض / نافد)
- ديون العملاء
- أفضل المنتجات مبيعاً
- المبيعات حسب الفئة
- اتجاه المبيعات الشهري
- ربح كشف الحساب (للمنتجات المسجلة فقط)

### 4.16 PDF والطباعة

يدعم إنشاء وطباعة:

- فاتورة بيع
- كشف حساب عميل
- قائمة أسعار
- كشف دفتر العميل (Ledger)

### 4.17 البنية التحتية المشتركة

- **Audit Logging** — تسجيل العمليات الحساسة
- **Caching** — تخزين مؤقت في الذاكرة لتسريع الاستعلامات
- **Validation** — التحقق من المدخلات
- **Security** — تنظيف المدخلات وتجزئة كلمات المرور
- **Error Handling** — معالجة أخطاء موحّدة (`AppException` / `Result`)

## 6. بنية المشروع (Architecture)

البناء يعتمد **Clean Architecture خفيفة** منظمة حسب الميزات (Feature-first):

```
lib/
├── main.dart
├── widgets/                 # ويدجت مشتركة (مثل Chatbot)
├── core/
│   ├── constants/           # نصوص وثوابت التطبيق
│   ├── database/            # DatabaseHelper ومولّد بيانات الاختبار
│   ├── di/                  # GetIt — حاوية الاعتماديات
│   ├── services/            # PDF، أمان، تدقيق، تحقق، كاش، تعريب، بحث ذكي، Chatbot
│   ├── theme/               # الألوان والثيم
│   └── utils/               # أدوات مساعدة
└── features/
    ├── auth/
    ├── dashboard/
    ├── products/
    ├── sales/
    ├── customers/
    ├── invoices/
    ├── expenses/
    ├── suppliers/
    ├── price_lists/
    ├── reports/
    ├── backup/
    └── settings/
```

كل ميزة تقريباً مقسّمة إلى:

- `domain/` — الكيانات وواجهات المستودعات
- `data/` — تنفيذ المستودعات وقاعدة البيانات
- `presentation/` — الصفحات، الـ Bloc، والويدجت

**تدفق التشغيل:** تسجيل الدخول → حالة مصادقة ناجحة → `DashboardPage` مع تبديل فوري بين التبويبات عبر `IndexedStack`.

## 7. الكيانات والجداول الرئيسية

### 7.1 كيانات المجال (Domain Entities)

- **User** — مستخدم (اسم، تجزئة كلمة المرور، دور، الاسم الكامل)
- **Product** — منتج ومخزون
- **Customer** — عميل وأرصدة
- **CustomerPayment** — دفعة عميل
- **CustomerLedger / CustomerLedgerEntry** — كشف حساب
- **Invoice** — فاتورة
- **SaleItem / SaleRecord / CartItem** — بنود البيع والسلة
- **Expense** — مصروف
- **Supplier / SupplierInvoice / SupplierPayment / SupplierAttachment**
- **PriceList / PriceListItem**
- **AccountLedgerProfitReport** — تقرير ربح كشف الحساب

### 7.2 جداول SQLite

`products`, `customers`, `users`, `invoices`, `sales`, `discounts`, `inventory_adjustments`, `cancelled_sales`, `expenses`, `additional_income`, `budget`, `store_settings`, `price_lists`, `price_list_items`, `suppliers`, `supplier_attachments`, `supplier_invoices`, `supplier_payments`, `customer_payments`, `audit_logs`

> بعض الجداول مثل `additional_income` و `budget` موجودة في قاعدة البيانات وقد لا تكون لها واجهة مستخدم كاملة بعد.

## 9. نقاط قوة البرنامج

- نظام متكامل (بيع + مخزون + محاسبة عملاء/موردين) في تطبيق واحد
- يعمل محلياً بدون إنترنت بعد التثبيت
- واجهة ثنائية اللغة مع RTL للعربية
- بحث ذكي ومساعد محلي يسرّعان العمل اليومي
- طباعة PDF للفواتير والكشوف وقوائم الأسعار
- نسخ احتياطي واستعادة لحماية بيانات المحل
- بنية كود منظمة (Features + BLoC) تسهّل التطوير المستقبلي
- أداء قاعدة بيانات محسّن (WAL، فهارس، كاش)

## 10. ملخص سريع

**Electrical Store** هو برنامج إدارة محل كهربائي لسطح مكتب Windows، مبني بـ **Flutter + BLoC + GetIt + SQLite**، ويغطي دورة العمل الكاملة: البيع، المخزون، العملاء والديون، الموردين، المصروفات، الفواتير، التقارير، قوائم الأسعار، النسخ الاحتياطي، والطباعة — مع دعم عربي/إنجليزي ومساعد ذكي محلي.
